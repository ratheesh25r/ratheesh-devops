1. What is Docker?

    Answer: Docker is an open-source platform used to automate the deployment, scaling, and management of applications inside lightweight, portable containers. Containers allow developers to package applications with all their dependencies, such as libraries and configurations, ensuring that the application can run consistently in any environment.

  2. What are containers in Docker?

    Answer: A container is a lightweight, standalone, executable package that includes everything needed to run a piece of software: code, runtime, system tools, libraries, and settings. Containers are isolated from each other and from the host system, providing a portable and consistent environment.


  3. What is the difference between a Docker container and a virtual machine?

    Answer:

    Containers: Share the host OS kernel, are lightweight, and are isolated from each other. They start quickly and have a small footprint.

    Virtual Machines (VMs): Run an entire OS along with the application, meaning they are much heavier and require more resources. Each VM includes its own OS, which results in slower startup times and higher resource usage.

  4. What is Docker Hub?

    Answer: Docker Hub is a cloud-based registry service provided by Docker. It allows users to store and share Docker images. It is the default registry for Docker images, where public images (such as official images) are stored and can be pulled and used by any Docker user.


  5. What are Docker Images?

    Answer: A Docker image is a read-only template used to create containers. Images contain the application and all dependencies required to run it. They are built from a Dockerfile, and you can store and share them on Docker Hub or other container registries.


  6. What is a Dockerfile?

    Answer: A Dockerfile is a text file that contains a set of instructions to build a Docker image. It specifies everything needed to set up the container environment, such as which base image to use, copying files, running commands, and setting up environment variables.

  7. Explain the Docker Build Process.

    Answer: The Docker build process involves creating a Docker image from a Dockerfile. Here's a simple breakdown:


    Dockerfile: The source file that defines the instructions.

    docker build: The command used to create a Docker image from the Dockerfile.

    Image: The resulting product from the build process that can be used to create containers.

  8. What is the difference between the CMD and ENTRYPOINT in a Dockerfile?

    Answer:


    CMD: Provides default arguments for the container's main process. It can be overridden by providing a different command when running the container.

    ENTRYPOINT: Specifies the main command to run when the container starts. The CMD arguments are passed as parameters to the ENTRYPOINT. ENTRYPOINT cannot be overridden unless you explicitly use

    --entrypoint in the docker run command.

  9. What is the purpose of the docker-compose tool?

    Answer: Docker Compose is a tool used for defining and running multi-container Docker applications. It allows you to define multiple containers, networks, and volumes in a single docker-compose.yml file.

    With a single command (docker-compose up), you can start all containers, networks, and volumes defined in the file.


  10. How can you list all Docker containers?

    Answer:


    To list all running containers: docker ps

    To list all containers (including stopped ones): docker ps -a

  11. What is the difference between docker run and docker exec?

    Answer:

    docker run: Starts a new container from an image and executes a command inside that container. docker exec: Executes a command inside a running container. It allows you to interact with the container

    after it has started.

  12. What is a Docker volume?

    Answer: A Docker volume is used to persist data outside of containers. Volumes are stored on the host filesystem and can be shared among multiple containers. Volumes ensure that data is retained even if the container is removed or stopped.


  13. How do you stop and remove a Docker container?

    Answer:


    To stop a container: docker stop <container_name_or_id> To remove a container: docker rm <container_name_or_id>

  14. What is the difference between docker stop and docker kill?

    Answer:

    docker stop: Gracefully stops the container by sending a SIGTERM signal, and after a timeout, it sends a SIGKILL signal.

    docker kill: Immediately stops the container by sending a SIGKILL signal without waiting for graceful shutdown.

  15. What is Docker Networking?

    Answer: Docker networking allows containers to communicate with each other, the host system, and the outside world. Docker provides different network types, such as:


    Bridge: Default network, containers on the same bridge network can communicate with each other.

    Host: Uses the host network stack, without isolation.

    Overlay: Enables communication between containers running on different Docker hosts.

    None: Disables networking for the container.

  16. How would you scale a Docker application?

    Answer: Docker applications can be scaled using Docker Compose or Docker Swarm. With Docker Compose, you can specify the number of container replicas for a service. In Docker Swarm, you can scale a service using the docker service scale command.

  17. What is Docker Swarm?

    Answer: Docker Swarm is Docker's native clustering and orchestration tool. It allows you to create and manage a cluster of Docker nodes and deploy services across them. It provides high availability, load balancing, and automatic scaling of applications.

  18. What are some advantages of Docker?

    Answer:

    Portability: Applications packaged in containers run the same way on any system. Isolation: Containers run in isolation, reducing conflicts between dependencies. Resource efficiency: Containers are lightweight compared to virtual machines.

    Scalability: Docker allows easy scaling and orchestration with tools like Docker Compose and Docker Swarm.

  19. What is the docker ps -a command used for?

    Answer: docker ps -a lists all containers (running and stopped). It provides information such as container ID, image, command, creation time, status, and port mappings.


  20. What is a multi-stage Docker build?

Answer: A multi-stage Docker build allows you to use multiple FROM statements in a Dockerfile. Each FROM represents a new stage in the build, which can help reduce the final image size by separating the build environment (with unnecessary build dependencies) from the runtime environment.


These questions and answers cover some of the most common topics related to Docker. They help interviewers assess both fundamental knowledge and practical expertise with Docker in real-world scenarios.
